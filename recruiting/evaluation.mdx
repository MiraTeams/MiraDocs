## Evaluation framework

When Mira returns search results, it doesn't just list names — it provides structured evaluation to help you make decisions quickly.

### Tiered recommendations

Candidates are grouped by match quality:

| Tier | Description |
| --- | --- |
| **Tier 1** | Extremely high match — direct experience with key requirements, strong background |
| **Tier 2** | High match — strong overlap with requirements, some gaps |
| **Tier 3** | Moderate match — relevant adjacent experience |

### Per-candidate analysis

For each recommended candidate, Mira provides:
- **Current role** — Where they work and what they do
- **Background** — Education, career trajectory, notable achievements
- **Highlights** — Why this candidate stands out for your specific role
- **Availability signals** — PhD graduation timing, recent role changes, etc.

## Asking for deeper analysis

You can ask Mira to go deeper on any candidate or comparison:

```
Compare the top 3 candidates side by side.
```

```
Which of these candidates would be easiest to recruit?
```

```
Draft an outreach message for Jiayi Weng highlighting why this role is a good fit.
```

## Priority recommendations

Mira provides actionable priority suggestions:

- **Immediate action** — Candidates ready to start, strongest backgrounds
- **Pipeline building** — Soon-to-graduate PhDs or candidates likely open to new roles
- **Strategic targets** — Senior leaders or researchers suitable for advisory or leadership roles
